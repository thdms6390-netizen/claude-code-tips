autoMode.hard_deny — Auto Mode 분류기보다 먼저 차단

Auto Mode 분류기가 허용하더라도 hard_deny 규칙에 걸리면 무조건 거부. 분류기 위에 올라타는 보안 레이어를 직접 정의할 수 있음.

NEW자동화보안2026.05.08 · v2.1.136
›

어떻게 쓰는 거야?

  • settings.autoMode.hard_deny에 차단할 분류기 규칙 패턴을 추가
  • Auto Mode 분류기가 판단하기 전에 먼저 평가돼서 매칭되면 무조건 거부
  • 민감한 파일 접근이나 특정 명령을 항상 차단하는 보안 강화 레이어로 활용 가능

worktree.baseRef — 워크트리 분기 기준 선택

fresh는 원격 기본 브랜치에서, head는 로컈 HEAD에서 브랜치를 딶다. 이제 워크트리 생성 기준을 직접 고를 수 있음.

NEWWorktree2026.05.07 · v2.1.133
›

설정 방법

  • worktree.baseRef: "fresh" — 항상 origin/<기본 브랜치>에서 새 워크트리를 만듦 (기존 기본값)
  • worktree.baseRef: "head" — 현재 로컈 HEAD 기준으로 브랜치를 딶다
  • --worktree 플래그, EnterWorktree 훅, 에이전트 격리 워크트리 모두에 적용됨
  • 로컈에서 직접 커밋을 쌓고 있는 상태라면 "head"로 두면 그 위에서 작업 가능

NEW자동화Hooks2026.05.07 · v2.1.133
›

  • 예: xhigh일 때만 전체 맰트 실행, low면 훅 동작 스�

CLAUDE_CODE_SESSION_ID — Bash 툴에 세션 ID 자동 주입

Bash 툴 서브프로세스 환경에 세션 ID가 자동으로 넘어와서, 훅(hooks)의 session_id와 바로 연결 가능.

NEW자동화2026.05.06 · v2.1.132
›

무슨 거냀면

  • Bash 툴로 실행하는 쉘 스크립트 안에서 $CLAUDE_CODE_SESSION_ID 환경변수를 바로 쓸 수 있음
  • 훅(hooks)에 넘어오는 session_id와 동일한 값 — 세션 추적·로깅에 활용 가능
  • 별도 설정 없이 자동으로 주입됨

CLAUDE_CODE_DISABLE_ALTERNATE_SCREEN — 네이티브 스크롤백 유지하기

풀스크린 alternate-screen 렌더러를 끄고 터미널 원래 스크롤백에 대화를 그대로 남길 수 있음.

NEW터미널 UI2026.05.06 · v2.1.132
›

사용 방법

  • CLAUDE_CODE_DISABLE_ALTERNATE_SCREEN=1 환경변수를 설정하면 풀스크린 렌더러가 꺼집
  • WSL·SSH·컨테이너의 alternate-screen이 불편한 환경에서 유용함
  • 대화 내용이 터미널 네이티브 스크롤백에 남아서 위로 올려볼 수 있음

Ctrl+R 히스토리 검색 — 전체 프로젝트 기본값 복원

Ctrl+R 히스토리 픽커가 다시 전체 프로젝트 기록을 기본으로 검색하도록 되돌아옴. Ctrl+S로 현재 프로젝트·세션으로 좋힐 수 있음.

NEW히스토리2026.5.6 · v2.1.129
›

변경 내용

  • v2.1.124~v2.1.128에서 현재 프로젝트 범위로만 검색하도록 바뀌었다가 원래대로 복원
  • Ctrl+R — 전체 프로젝트·세션에서 과거 프롬프트 검색 (기본값)
  • Ctrl+S — 현재 프로젝트나 세션으로 범위 좋히기

--plugin-url 플래그 — URL에서 플러그인 직접 로드

플러그인 .zip 파일을 URL로 지정해서 현재 세션에 바로 불러올 수 있음. 로컈 설치 없이 URL만 있으면 됨.

NEW플러그인2026.5.6 · v2.1.129
›

사용법

  • claude --plugin-url https://example.com/myplugin.zip 형태로 실행
  • 로컈에 설치하지 않고 URL에 있는 .zip 플러그인을 세션에 즉시 로드하는 거
  • 팀원과 플러그인 공유하거나 테스트할 때 설치 없이 바로 써볼 수 있음
