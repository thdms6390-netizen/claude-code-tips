Bedrock 서비스 티어 선택 — AWS 요청 우선순위 직접 지정

환경변수 하나로 Bedrock 호출 시 default·flex·priority 중 원하는 티어를 골라서 보낼 수 있음.

NEWAWSBedrock2026.4.29 · v2.1.122
›

이게 왜 필요해?

AWS Bedrock은 요청 처리 우선순위를 티어로 관리함. 기존엔 기본값만 쓸 수 있었는데, 이제 ANTHROPIC_BEDROCK_SERVICE_TIER를 설정하면 X-Amzn-Bedrock-Service-Tier 헤더로 자동 전달됨.

  • default — 기본 처리 (변경 없음)
  • flex — 유연한 처리 (비용 최적화)
  • priority — 우선 처리 (속도 우선, 엔터프라이즈용)

설정법

# .env 또는 셸 설정에 추가
ANTHROPIC_BEDROCK_SERVICE_TIER=priority

# 또는 실행 시 인라인으로
ANTHROPIC_BEDROCK_SERVICE_TIER=flex claude

prUrlTemplate — PR 배지 링크 바꾸기

기본적으로 하단 PR 배지는 github.com 링크를 가리키는데, 사내 GitLab이나 Gerrit 같은 커스텀 코드리뷰 시스템을 쓴다면 prUrlTemplate으로 URL 패턴을 직접 지정할 수 있어.

}

CLAUDE_CODE_HIDE_CWD — 시작 로고에서 경로 숨기기

Claude Code를 실행하면 시작 로고에 현재 작업 디렉터리가 표시되는데, 화면 공유하거나 스크린샷 찍을 때 경로가 노출되기 싫다면 이 환경변수로 숨길 수 있어.

deniedDomains 설정 — 허용 도메인 안에서도 특정 주소는 차단

샌드박스 네트워크 설정에 예외 차단 기능이 생겼어. allowedDomains에 와일드카드로 넓게 열어놨어도, 특정 도메인만 콕 집어서 막을 수 있음.

NEW보안자동화2026. 4. 18. · v2.1.113
›

이게 왜 필요해?

allowedDomains: ["*.example.com"] 처럼 와일드카드로 넓게 허용했을 때, 그 안에 있는 특정 서브도메인은 접근 못 하게 막고 싶을 때 쓰는 거. 예를 들어 내부 관리자 API 엔드포인트만 차단하는 식으로 활용 가능.

설정 방법 (settings.json)

# ~/.claude/settings.json
{
  "sandbox": {
    "network": {
      "allowedDomains": ["*.mycompany.com"],
      "deniedDomains": ["admin.mycompany.com", "internal.mycompany.com"]
    }
  }
}

우선순위

deniedDomains가 allowedDomains보다 우선 적용돼. 즉, 허용 목록에 들어있어도 차단 목록에 있으면 무조건 막힘. 세밀한 네트워크 제어가 필요한 팀 환경에서 유용함.

PreCompact 훅 — 컴팩트 직전에 끼어들기

대화가 압축되기 전에 훅을 실행할 수 있음. 필요하면 컴팩트 자체를 막는 것도 가능.

NEWHooks2026. 4. 14. · v2.1.105
›

이게 왜 유용함?

Claude Code가 컨텍스트가 길어지면 자동으로 /compact를 돌리는데, 그 타이밍을 훅으로 가로챌 수 있음. 예를 들어 중요한 작업 중간에 갑자기 압축되는 걸 막거나, 압축 전에 현재 상태를 파일로 저장하거나, 조건에 따라 아예 블록할 수 있음.

블록 방법 두 가지

  • 훅 스크립트에서 exit code 2로 종료하면 컴팩트가 중단됨
  • stdout으로 {"decision":"block"}을 반환해도 동일하게 블록됨

설정 예시

# ~/.claude/settings.json
{
  "hooks": {
    "PreCompact": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "node ~/scripts/before-compact.js"
          }
        ]
      }
    ]
  }
}

스크립트에서 블록하는 법

# exit code 2로 블록
exit 2

# 또는 JSON으로 블록
echo '{"decision":"block"}'

어떻게 동작함?

  • 스킬 실행 중 에러 로그를 실시간 감시하다가 이상 감지 시 Claude에게 알림
  • 빌드/테스트 스킬 invoke 시 CPU·메모리 사용량 모니터 자동 시작
  • 세션 내내 특정 파일 변경을 감지해서 자동으로 후속 작업 트리거

      "trigger": "session_start"

›

이게 뭔데?

기존엔 MCP 툴이 반환하는 결과 크기에 제한이 있어서, DB 스키마나 긴 파일 목록 같은 데이터가 중간에 잘렸음. 이제 MCP 툴 응답의 _meta에 anthropic/maxResultSizeChars 어노테이션을 넣으면 최대 500K까지 늘릴 수 있음.

  "content": "...긴 DB 스키마...",
  "_meta": {
    "anthropic/maxResultSizeChars": 500000
  }
}

  • DB 전체 스키마를 한 번에 Claude에게 넘길 때
  • 대형 JSON 설정 파일을 MCP로 읽어올 때
  • 툴 결과가 잘려서 Claude가 엉뚱한 답을 냈던 경우

PermissionDenied 훅 — 거절당했을 때 재시도 제어하기

자동 모드에서 권한 거절이 뜰 때 훅으로 가로채서 "다시 해봐"라고 모델에게 알려줄 수 있음.

NEW훅/자동화자동화2025.3.31 · v2.1.88
›

어떻게 동작하냐면

Claude Code의 auto 모드 classifier가 특정 작업을 거절(deny)했을 때, 이제 PermissionDenied 훅이 발동됨. 이 훅에서 {retry: true}를 반환하면 모델한테 "이거 다시 시도해도 됨"이라고 신호를 보낼 수 있는 거야.

훅 설정 예시

# .claude/hooks.js (예시)
export const PermissionDenied = async ({ tool, reason }) => {
  // 특정 조건일 때만 재시도 허용
  if (tool === 'read_file' && reason === 'sandbox') {
    return { retry: true };
  }
  // 기본은 그냥 거절 유지
  return {};

이게 왜 좋냐면

기존엔 훅이 모든 상황에서 무조건 실행됐는데, 이제 if 필드로 조건을 걸 수 있음. 예를 들어 git 관련 명령어를 실행할 때만 훅이 발동되게 하거나, 특정 파일 작업에서만 동작하게 제한할 수 있는 거야. 쓸데없이 훅이 자꾸 뜨는 거 방지됨.

설정 예시 (settings.json)

# git 명령어 실행할 때만 이 훅이 발동됨
{
  "hooks": {
    "PreToolUse": [
      {
        "if": "Bash(git *)",
        "command": "echo 'git 명령어 감지!'"
      }
    ]
  }
}

조건 문법

권한 규칙 문법을 그대로 사용함. Bash(git *), Write(src/**) 같은 식으로 툴 이름 + 패턴으로 쓰면 됨.

Ultrathink 복귀 — "ultrathink" 한 마디로 깊게 생각

프롬프트에 "ultrathink" 쓰면 다음 턴만 high effort로 실행. 어려운 문제에 집중.

Opus 4.62026.03.04 · v2.1.68
›

effort가 뭔데?

  • Low — 빠르게 답변, 간단한 질문용
  • Medium — 기본값 (Opus 4.6). 대부분의 작업에 적합
  • High — 깊이 생각. 복잡한 버그, 아키텍처 설계 등

사용법

# 프롬프트에 키워드 포함
ultrathink 이 버그 원인 찾아줘

# 또는 effort 레벨 직접 변경
/effort high

# Opus 4.6는 기본이 medium effort
💡 "ultrathink"은 해당 턴만 high effort. 이후는 다시 기본값으로 돌아갑니다.
